Product Requirements Document (PRD): DBA-Genie AI Agent1. Document OverviewProject Name: DBA-GenieVersion: 1.0.0Status: DraftPrimary Objective: Create an AI-driven Database Administrator (DBA) agent that translates natural language and documents into executable database structures and queries.2. Product VisionTo empower developers and business analysts to manage database schemas and extract data insights using natural language, reducing the friction of writing manual SQL and navigating complex table relationships.3. Functional Requirements3.1 Requirement Ingestion & ExtractionFR-1: Multi-format Input: The system must accept raw text input or file uploads (.txt, .pdf, .docx).FR-2: AI Schema Extraction: The AI agent must identify entities, data types, and primary/foreign key relationships from the input.FR-3: Script Generation: The agent must generate a valid SQL CREATE TABLE script based on the extracted schema.3.2 Database Interaction (The "Agent" Layer)FR-4: Live Command Execution: The agent must be able to execute CREATE, ADD (ALTER), and DELETE (DROP) commands on a target database.FR-5: Schema Exploration: Given a connection string, the agent must fetch a list of tables, views, and column metadata.FR-6: Natural Language to Query (NL2SQL): The user can ask a question (e.g., "Who are my top 10 customers?"), and the agent generates the corresponding SELECT statement based on the known schema.3.3 Model & ConfigurationFR-7: LLM Provider Toggle: The system must support switching between:OpenAI (GPT-4o)Google Gemini (1.5 Pro/Flash)Anthropic (Claude 3.5 Sonnet)FR-8: Connection Management: Secure storage (session-based) of database connection strings.4. Technical StackComponentTechnologyReasonFrontendReactJS (Hooks/Context API)High interactivity for schema visualization and chat.BackendC# .NET Framework 4.6.2 MVCAlignment with existing enterprise infrastructure.DatabaseSQL Server / PostgreSQLTarget environments for the DBA agent.AI IntegrationREST API / HttpClientNecessary for .NET 4.6.2 compatibility with modern LLM APIs.5. System Architecture & Workflow5.1 Backend Logic (.NET 4.6.2)Since .NET 4.6.2 is an older framework, the backend will act as a Smart Proxy:Request: React sends the requirement + selected LLM type.Prompt Engineering: The C# service wraps the user input in a system prompt that defines the "DBA Persona."API Call: Use System.Net.Http.HttpClient to POST to OpenAI/Gemini/Anthropic endpoints.SQL Parsing: A utility class validates the returned SQL for basic safety (e.g., blocking SHUTDOWN or GRANT commands).5.2 Frontend Flow (React)Dashboard: Connect to a DB or upload a requirement.Agent Chat: A chat interface where the user sees the AI "thinking" and "executing."Schema View: A visual representation of tables generated or explored.6. Safety & GuardrailsG-1: Destructive Action Confirmation: Any DROP or DELETE command must trigger a manual "Are you sure?" confirmation in the React UI.G-2: Query Limits: All generated SELECT queries will automatically have a TOP 100 or LIMIT 100 appended to prevent system memory crashes.G-3: Credential Security: Connection strings should never be logged or stored in plain text on the server-side logs.7. Success MetricsSchema Accuracy: The AI correctly identifies 90% of relationships (One-to-Many, etc.) from a standard business document.Execution Rate: Successfully running generated scripts on the first attempt without syntax errors.Latency: Receiving a generated SQL script from the LLM in under 8 seconds.

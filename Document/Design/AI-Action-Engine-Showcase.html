What Administrators Configure

Skill Library

Prompt Template Management

Each AI capability begins with an AppAISkill record — a named system prompt the engine resolves at runtime. Multiple skills can share the same engine.

// AI Skill Management UI → save a skill { "Name": "FabricBomExtract", "SkillContent": "You are a fashion industry expert.\nAnalyze the input and extract a\ncomplete fabric Bill of Materials.\nReturn only valid JSON:\n{ \"rows\": [...], \"warnings\": \"\" }" }
Command Action Editor

Input / Output Binding

A CommandActionButton with ActionType = 56 on any form activates the engine. The visual editor generates this configuration automatically:

{ "skillName": "FabricBomExtract", "inputBindings": [{ "fieldName": "ProductImage", "inputType": "image" }], "outputBindings": [{ "outputKey": "rows", "targetType": "child_grid", "targetName": "FabricBomLines" }] }
Auto-Map Engine

AI Keys → Database Columns

The runtime resolves AI-returned friendly keys to actual database column names using the form's own field metadata — no explicit column mapping configuration needed.

// AI returns friendly keys { "materialName": "Cotton Twill", "fiberComp": "80% Cotton" } // Auto-map resolves via field metadata { "Fabric_Name": "Cotton Twill", "Fiber_Composition": "80% Cotton" } // Lookup built from transactionExDto at runtime buildDbNameLookup(transactionExDto, unitId)
Multi-Provider Support

One Engine, Any Provider

Provider selection is a single configuration value in appsettings.json. The engine handles all protocol differences — message formats, image encoding, system prompt placement — transparently.

Anthropic Claude claude-sonnet-4 · claude-opus-4
Vision Text

System prompt in top-level system field. Images as base64 content blocks with source.type = "base64".

OpenAI GPT-4o gpt-4o · gpt-4-turbo
Vision Text

System role message. Images as image_url blocks with data URI data:mime;base64,….

Google Gemini gemini-2.0-flash · gemini-1.5-pro
Vision Text

System prompt via system_instruction. Images as inline_data parts alongside text parts.

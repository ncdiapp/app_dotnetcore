How to Build a Report — Order & Order Detail

A step-by-step walkthrough covering all three design modes, token insertion, table generation, and live preview.
Three Modes at a Glance
</> Code Write HTML + CSS directly. Full control. Type {{ for token autocomplete.
△ Visual Read-only preview of rendered output with real data. Drag tokens to insert into the layout.
✎ Design Drag-and-drop WYSIWYG editor. Arrange blocks visually, then switch to Code to add tokens.

Before You Start

The designer needs a stored procedure that returns order data. Your DBA creates it once; you reference it by name. 

-- Minimal SP contract for an Order Detail report
CREATE PROCEDURE sp_OrderDetailReport
    @MainReferenceId    INT,
    @MasterReferenceId  INT = NULL,
    @ExtraParams        NVARCHAR(MAX) = NULL
AS BEGIN
    -- RS0: order header scalars (one row)
    SELECT o.OrderNumber, c.CustomerName, o.OrderDate,
           o.Status, o.Currency, o.OrderTotal
    FROM Orders o JOIN Customers c ON c.CustomerId = o.CustomerId
    WHERE o.OrderId = @MainReferenceId

    -- RS1: order line items (repeating rows)
    SELECT ol.Sku, ol.Description, ol.Quantity, ol.UnitPrice, ol.Total
    FROM OrderLines ol
    WHERE ol.OrderId = @MainReferenceId
    ORDER BY ol.LineNumber
END
Step-by-Step Walkthrough
1

Open the Report Designer

Go to Administration → Report Management. Find your report and click the designer icon, or create a new report and open it.

The designer opens full-screen. You'll see the toolbar (page size, margins, preview, save), a left panel (Blocks / Tokens tabs), and the editor area with the Code / Visual / Design toggle.

2

Configure the Data Source

Click the Sources button in the toolbar to open the Data Sources modal.

Toolbar
🗄 1 Source ✎ Size: A4 ▼ portrait ▼ Margin: 15

In the modal, set the source Name to order and the SP Name to sp_OrderDetailReport. Click Apply & Load Tokens.

⚠
Source name matters The name you give the source becomes the token prefix. Source named order produces tokens like {{order.OrderNumber}}. Choose a short, memorable name with no spaces.
3

Verify Your Tokens

Click the Tokens tab in the left panel. The designer automatically refreshes token discovery when you switch to this tab.

You should see two groups:

Tokens panel — expected result
🗄 ORDER
{{order.OrderNumber}} {{order.CustomerName}} {{order.OrderDate}} {{order.Status}} {{order.Currency}} {{order.OrderTotal}} ☰ #each order_rs1   ↳ Sku   ↳ Description   ↳ Quantity   ↳ UnitPrice   ↳ Total

The orange #each token marks the repeating section (RS1 lines). Indented tokens below it are the fields inside each row.

4

Start from a Template (optional)

Click the Blocks tab → Browse templates… button. The gallery shows HTML thumbnail previews of each starter.

For an order report, choose Order Detail. This loads a complete starter layout — you'll replace the placeholder tokens with your actual order. tokens in the next steps.

ⓘ
If none of the starters match your layout, choose Start blank and build it from scratch using blocks in Step 5.
5

Build the Layout in Design Mode

Click Design in the mode toggle (purple button). The GrapeJS visual editor loads.

⚠
Tokens in Design mode GrapeJS shows token placeholders as plain text (e.g. {{header.Title}}). This is correct — they are replaced with real data at preview/PDF time. Do not delete them.
6

Insert Tokens in Code Mode

Switch to Code mode. The HTML from your Design layout appears in the Monaco editor. Now replace the placeholder text with your actual tokens.

Method A — autocomplete: Place cursor where you want a token, type {{, and pick from the dropdown.

Method B — click to insert: Click any token in the Tokens tab — it inserts at the cursor position.

Method C — drag & drop: Drag a token from the left panel and drop it onto any line in the editor.

Example — replace the Report Header block's placeholder fields:

<!-- Before -->
<h1 style="...">{{header.Title}}</h1>
<div style="...">{{header.Field1}} | {{header.Field2}}</div>

<!-- After -->
<h1 style="...">{{order.OrderNumber}}</h1>
<div style="...">
  {{order.CustomerName}}  | 
  {{order.OrderDate}}  | 
  {{order.Status}}
</div>

8

Preview with a Live Record

Switch to Visual mode. In the toolbar bar, find the Preview record # field — enter a real Order ID from your database.

Visual mode bar
△ Preview — A4 portrait Preview record # 1042 ↻ Refresh

Click Refresh (or wait for the auto-refresh). The iframe shows your template populated with real order data from the SP.

If fields show as {{order.OrderNumber}} instead of a value, check: (1) the SP name is correct, (2) the record ID exists, (3) the token prefix matches the source name.

Use PDF Preview in the toolbar to see the final print output — this runs Playwright and shows the exact PDF that will be generated.

9

Fine-tune Styling in Code Mode

Switch back to Code mode for precise styling adjustments. Common tweaks:

/* Add a totals row after the #each block */
    {{/each}}
    <tr style="font-weight:bold;background:#f0f4ff;">
      <td colspan="4" style="text-align:right;padding:6px 10px">
        Order Total:
      </td>
      <td style="padding:6px 10px">{{order.OrderTotal}}</td>
    </tr>

/* Conditional display — only show when field has a value */
{{#if order.Notes}}
  <p style="font-size:11px;color:#666;">Notes: {{order.Notes}}</p>
{{/if}}
10

Save

Click Save in the toolbar. The template is stored to the database — page size, orientation, margin, data source config, and HTML are all saved together.

✓
Done. The report is now available for printing from any Order form. Users click the Report button → select this report → Save as PDF.

Tips & Common Mistakes
{{
Token autocomplete: type {{ in Code mode The editor shows a dropdown of all available tokens the moment you type {{. No need to type the full name — arrow keys to navigate, Enter to insert.
✎
Design mode is for layout, Code mode is for tokens GrapeJS is great for dragging blocks into position. But token placeholders ({{order.X}}) are easiest to edit in Code mode where you have full text control.
✓
Field names inside #each have no prefix Outside a loop: {{order.OrderNumber}}. Inside {{#each order_rs1}}: just {{Sku}}, {{Quantity}}. The prefix is implied by the loop context.
!
SP not returning tokens? Check the source name The source Name in the Data Sources modal (e.g. order) must match what you reference in templates. The SP Name (e.g. sp_OrderDetailReport) is what the backend calls — these are two separate fields.
A4
Set page size before designing The canvas preview in Visual mode respects the page size setting. Set Size + Orientation in the toolbar before you start so you know exactly what space you have available.
↻
Always verify with PDF Preview before saving HTML Preview is fast but doesn't paginate. PDF Preview runs the full Playwright renderer — use this to catch content that overflows the page before sending to users.
